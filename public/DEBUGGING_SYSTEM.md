# 🐛 종합 디버깅 시스템 가이드

## 📋 개요
무료 AI API를 활용한 최상위 수준의 종합 디버깅 시스템을 구축했습니다.

---

## ✨ 주요 기능

### 1. **실시간 콘솔 로그 모니터링**
- ✅ 모든 `console.log`, `console.warn`, `console.error` 자동 수집
- ✅ 타임스탬프 및 파일 위치 추적
- ✅ 레벨별 필터링 (error, warn, info, log)
- ✅ 검색 기능으로 빠른 로그 찾기
- ✅ 스택 트레이스 포함

### 2. **브레이크포인트 관리**
- ✅ 파일별 브레이크포인트 설정
- ✅ 조건부 브레이크포인트 (조건이 참일 때만 중단)
- ✅ 브레이크포인트 활성화/비활성화
- ✅ 히트 카운트 추적
- ✅ 원클릭 추가/제거

### 3. **네트워크 요청 디버깅**
- ✅ 모든 fetch 요청 자동 가로채기
- ✅ 요청/응답 헤더 및 본문 확인
- ✅ 응답 시간 측정
- ✅ HTTP 상태 코드 확인
- ✅ 에러 요청 자동 표시

### 4. **AI 기반 자동 버그 수정**
- ✅ 무료 AI API 활용 (Hugging Face, Replicate, OpenRouter)
- ✅ 에러 메시지 및 코드 분석
- ✅ 자동 수정 코드 제안
- ✅ 신뢰도 점수 제공
- ✅ 수정 설명 및 이유 제공

### 5. **디버깅 세션 관리**
- ✅ 여러 디버깅 세션 생성 및 관리
- ✅ 세션별 브레이크포인트 및 로그 저장
- ✅ 세션 재사용 및 공유
- ✅ 세션 히스토리 추적

## 🎯 사용 방법

### 1. 디버깅 패널 열기
사이드바 → **"디버깅"** 탭 클릭

### 2. 콘솔 로그 확인
1. "콘솔" 탭 선택
2. 실시간 로그 확인
3. 필터로 에러만 보기:
   - 레벨 선택: "에러"
   - 검색: "undefined" 입력

### 3. 브레이크포인트 설정
1. "브레이크포인트" 탭 선택
2. "추가" 버튼 클릭
3. 파일 경로와 라인 번호 입력
4. (선택) 조건 입력 예: `count > 10`

### 4. 네트워크 요청 디버깅
1. "네트워크" 탭 선택
2. 모든 API 요청 자동 수집 확인
3. 요청 클릭하여 상세 정보 확인
4. 에러 요청은 빨간색으로 표시

### 5. AI 버그 수정
1. "AI 디버깅" 탭 선택
2. 에러 메시지 입력:
   ```
   TypeError: Cannot read property 'map' of undefined
   ```
3. 코드 입력:
   ```javascript
   const items = undefined;
   return items.map(item => item.name);
   ```
4. "AI로 버그 수정하기" 클릭
5. 수정된 코드 확인 및 적용

### 6. 디버깅 세션 관리
1. "세션" 탭 선택
2. "세션 시작" 버튼 클릭
3. 디버깅 작업 수행
4. 세션 저장 및 나중에 재사용

## 🔧 고급 기능

### 조건부 브레이크포인트
조건을 입력하면 해당 조건이 참일 때만 실행이 중단됩니다.

예시:
- `count > 100`: count가 100 초과일 때
- `user && user.role === 'admin'`: 관리자일 때
- `error !== null`: 에러가 있을 때

### AI 디버깅 팁
1. **명확한 에러 메시지 제공**: AI가 정확히 분석할 수 있도록 전체 에러 메시지를 복사해 붙여넣으세요.

2. **주변 코드 포함**: 문제가 발생한 함수 전체를 포함하면 더 정확한 수정을 받을 수 있습니다.

3. **컨텍스트 설명**: React 컴포넌트, API 호출 등 추가 정보를 에러 메시지와 함께 입력하세요.

### 네트워크 디버깅
- **요청 본문 확인**: POST/PUT 요청의 데이터 확인
- **응답 분석**: 서버 응답 데이터 구조 파악
- **성능 측정**: 각 요청의 소요 시간 확인
- **에러 추적**: 4xx, 5xx 에러 자동 표시

## 🤖 AI 디버깅 작동 원리

### 지원 AI 서비스
1. **Hugging Face** (무료)
   - CodeLlama 모델 사용
   - 코드 분석 및 수정

2. **Replicate** (무료 티어)
   - 다양한 오픈소스 모델
   - 빠른 응답 시간

3. **OpenRouter** (무료 모델)
   - Mistral, Llama 등
   - 다중 모델 지원

### 처리 과정
1. **코드 분석**: 입력된 코드를 AI가 분석
2. **에러 패턴 인식**: 에러 메시지와 코드를 매칭
3. **수정 제안**: 안전하고 효율적인 수정 코드 생성
4. **신뢰도 계산**: 수정 제안의 정확도 평가
5. **설명 제공**: 왜 이렇게 수정해야 하는지 설명

## 💡 사용 팁

### 효율적인 디버깅 워크플로우
1. **세션 시작** → 디버깅 시작 전 세션 생성
2. **브레이크포인트 설정** → 중요한 지점에 브레이크포인트
3. **콘솔 모니터링** → 에러 발생 시 즉시 확인
4. **AI 수정** → 복잡한 버그는 AI에게 수정 제안 요청
5. **세션 저장** → 나중에 참고할 수 있도록 저장

### 자주 발생하는 문제 해결

#### 문제: 로그가 너무 많아서 찾기 어려움
**해결**: 필터 사용
- 레벨: "에러"만 선택
- 검색: 키워드 입력

#### 문제: 네트워크 요청이 보이지 않음
**해결**: 
- 페이지를 새로고침
- 네트워크 탭에서 요청 확인

#### 문제: AI 수정이 정확하지 않음
**해결**:
- 더 많은 컨텍스트 제공
- 전체 함수 코드 포함
- 에러 메시지 전체 복사
