# 구현 우선순위 및 계획

## 🎯 핵심 원칙
1. **실제 구동 가능한 영역에서만 진행**
2. **단계별 우선순위 명확히**
3. **즉시 사용 가능한 템플릿 확보**

## 📊 우선순위 재정리

### Phase 1: 즉시 완성 가능 (1-2일)
✅ **완료된 항목**
- AI 해시태그 생성기
- 가격 정책 업데이트
- 기본 DB 구조

🔨 **즉시 구현 (우선순위 1)**
1. **템플릿 시스템 구축** ⭐ 최우선
   - 템플릿 데이터 구조
   - 템플릿 라이브러리 페이지
   - 템플릿 적용 기능
   - 초기 템플릿 50개 확보 (점진적 확장)

2. **프로젝트 상세 화면** ⭐
   - 5단계 콘텐츠 제작 UI
   - 단계별 재생성 버튼
   - 진행 상태 표시

3. **결제 시스템 기본 구조** ⭐
   - Stripe 통합 (최소 기능)
   - 플랜 업그레이드 플로우

### Phase 2: 단기 완성 가능 (3-5일)
4. **인증 시스템**
   - 간단한 회원가입/로그인
   - 세션 관리

5. **다른 관리자 기능 유료화**
   - 각 기능에 플랜 제한 적용
   - 업그레이드 프롬프트 통합

### Phase 3: 중장기 (나중에)
6. SNS 자동 포스팅 스케줄러 (API 연동 필요)
7. 숏폼 콘텐츠 생성기 (추가 개발 필요)
8. 트렌딩 주제 추천 (API 연동 필요)

## 🎨 템플릿 전략

### 템플릿 개수
- **초기**: 50개 (실제 구현 가능)
- **1단계 확장**: 100개
- **목표**: 200-300개 (1천개는 후속 단계)

### 템플릿 카테고리
1. **블로그 포스트** (20개)
   - 기술 블로그
   - 리뷰 블로그
   - 가이드 블로그
   - 뉴스 블로그

2. **유튜브 스크립트** (15개)
   - 리뷰 영상
   - 튜토리얼
   - 브이로그
   - 교육 영상

3. **SNS 게시물** (10개)
   - 제품 홍보
   - 이벤트 안내
   - 팁/노하우
   - 질문형

4. **인스타그램 캡션** (5개)
   - 제품 소개
   - 일상 공유
   - 영감 주는 글

### 템플릿 구조
```typescript
interface ContentTemplate {
  id: string;
  title: string;
  category: string;
  contentType: string;
  platform: string;
  description: string;
  example: string; // 예시 콘텐츠
  structure: {
    sections: string[]; // 섹션 구조
    tips: string[]; // 작성 팁
  };
  tags: string[];
}
```
