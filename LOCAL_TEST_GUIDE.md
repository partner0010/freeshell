## 🔧 문제 해결

### Ollama가 연결되지 않을 때

#### 1. Ollama 서비스 확인
```bash
# Windows
# 작업 관리자에서 "Ollama" 프로세스 확인

# Mac/Linux
ps aux | grep ollama
```

#### 2. Ollama 수동 시작
```bash
ollama serve
```

#### 3. 포트 확인
브라우저에서 접속:
```
http://localhost:11434
```

응답이 오면 정상 작동 중입니다!

#### 4. 모델 확인
```bash
ollama list
```

`llama3.1:8b`가 목록에 있는지 확인하세요.

### Monaco Editor가 로드되지 않을 때

#### 1. 패키지 설치 확인
```bash
npm install @monaco-editor/react monaco-editor
```

#### 2. 브라우저 캐시 삭제
- Ctrl+Shift+R (강력 새로고침)
- 또는 개발자 도구 → Network → "Disable cache" 체크

#### 3. 콘솔 에러 확인
- F12 → Console 탭
- 에러 메시지 확인

### AI 응답이 느릴 때

#### 1. 모델 크기 확인
```bash
ollama list
```

더 작은 모델 사용:
```bash
ollama pull phi3  # 매우 빠름
```

#### 2. Groq API 사용
Groq API가 더 빠를 수 있습니다 (이미 설정됨).

## 📊 성능 비교

### 로컬 테스트 환경
- **Groq API**: ~1-2초 (가장 빠름) ⚡
- **Ollama (llama3.1:8b)**: ~3-5초 (로컬, 무료)
- **Ollama (phi3)**: ~1-2초 (매우 빠름, 경량)

### 추천 설정
1. **가장 빠름**: Groq API (이미 설정됨)
2. **완전 무료**: Ollama (llama3.1:8b)
3. **균형**: Groq + Ollama (자동 전환)
